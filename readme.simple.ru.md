# DeepONet для финансов -- Простое объяснение

## Что такое DeepONet?

Представьте, что вы повар, знающий тысячи рецептов. Обычная нейронная сеть -- это как заучивание одного конкретного рецепта: "При этих точных ингредиентах (входах) приготовь это точное блюдо (выход)." Если кто-то слегка изменит ингредиенты, вам придётся учить совершенно новый рецепт с нуля.

**DeepONet -- это как изучение самого искусства кулинарии.** Вместо запоминания одного рецепта вы учите *процесс* превращения любого набора ингредиентов в готовое блюдо. Дали итальянские ингредиенты? Получится итальянская кухня. Дали японские ингредиенты? Получится японская кухня. Один "кулинарный оператор" справляется со всем.

В математических терминах DeepONet обучает **операторы** -- отображения из функций в функции -- а не отображения из чисел в числа.

## Аналогия с кухней двух поваров

DeepONet состоит из двух подсетей, которые работают вместе как два повара на кухне:

### Повар 1: Branch-сеть (Дегустатор)

Этот повар пробует сырые ингредиенты и создаёт мысленный профиль блюда:

```
Сырые ингредиенты            Мысленный профиль
(входная функция)      →     (латентный код)

"Хм, эта мука               [сладкий, воздушный,
 мелкого помола, масло        насыщенный, деликатный,
 несолёное, сахар             с нотками ванили]
 пудра..."
```

Branch-сеть читает **всю входную функцию** (поверхность волатильности, историю цен или экономические индикаторы) и сжимает её в компактное представление.

### Повар 2: Trunk-сеть (Сервировщик)

Этот повар решает, как подать блюдо -- где каждый элемент располагается на тарелке:

```
Позиция на тарелке           Правило сервировки
(точка запроса)        →     (базисные функции)

"В позиции (3см,             [мазок, завиток,
 5см) на тарелке..."          полоска, украшение]
```

Trunk-сеть читает **точку запроса** (конкретный страйк и срок для опциона, или конкретный момент времени для прогноза) и определяет пространственную структуру выхода.

### Финальное блюдо

Результат создаётся объединением работы обоих поваров:

```
Блюдо = Σ (профиль Дегустатора) × (правила Сервировщика) + приправа(смещение)
```

Это скалярное произведение означает: "Примени понимание ингредиентов Дегустатором, используя знания Сервировщика о том, куда что разместить."

## Почему это мощно для финансов?

### Аналогия с синоптиком

Представьте синоптика:

**Подход стандартной НС**: Обучить одну модель для "дождливых дней", другую для "солнечных дней" и третью для "штормовых дней." Каждая модель работает только для своего погодного паттерна.

**Подход DeepONet**: Обучить одну модель, которая понимает *как работает погода*. Дайте ей сегодняшние атмосферные условия (входная функция), и она скажет вам температуру, влажность или скорость ветра в любом месте и в любое будущее время (точка запроса).

### Реальные финансовые примеры

#### Ценообразование опционов -- Аналогия со страхованием

Представьте ценообразование опционов как страхование:

- **Входная функция**: "Ландшафт рисков" (поверхность волатильности) -- как карта, показывающая насколько опасны разные дороги
- **Точка запроса**: Конкретный страховой полис (страйк K, дата экспирации T) -- как вопрос "сколько стоит страховка на Трассу М4 на 3 месяца?"
- **Выход**: Страховая премия (цена опциона)

Один DeepONet обучает *весь процесс страхового ценообразования*, а не один полис. Изменился ландшафт рисков? Не нужно переобучение -- оператор естественно производит новые цены.

#### Кривая доходности -- Аналогия с рекой

Представьте кривую доходности как реку, текущую через экономику:

- **Входная функция**: Экономические условия (рост ВВП, инфляция, занятость) -- как осадки, рельеф и температура, влияющие на реку
- **Точка запроса**: Как далеко по течению вы смотрите (срок облигации: 1 год, 5 лет, 30 лет)
- **Выход**: Скорость потока в этой точке (процентная ставка при данном сроке)

DeepONet обучает тому, как экономический "дождь" превращается в форму "реки" процентных ставок.

## Physics-Informed DeepONet -- Добавление правил

### Аналогия с настольной игрой

Представьте обучение шахматам:

- **Стандартное обучение**: Показать тысячи партий и дать выучить паттерны (чистые данные)
- **Физически-информированное обучение**: Показать партии И научить правилам (данные + физика)

PI-DeepONet добавляет финансовые "правила" (как PDE Блэка-Шоулза) как дополнительные ограничения при обучении:

1. Модели нужно меньше примеров для обучения
2. Предсказания всегда удовлетворяют финансовым законам (безарбитражность, паритет пут-колл)
3. Лучше работает вне диапазона обучающих данных

```
Обычный DeepONet:        "Я видел 10 000 цен опционов, вот моя оценка"
PI-DeepONet:             "Я видел 1 000 цен опционов И я знаю физику
                          ценообразования опционов. Вот более точный ответ."
```

## Многоточностный DeepONet -- Подход с черновиком

### Аналогия с художником

Представьте портретиста:

1. **Черновой набросок** (Блэк-Шоулз): Быстрый, дешёвый, приблизительно верный. Как использование простой формулы для грубой оценки цены опциона.

2. **Детальная картина** (Monte Carlo по Хестону): Красивая и точная, но занимает часы. Как запуск миллионов симуляций.

3. **Многоточностный подход**: Начать с чернового наброска, затем обучить коррекцию, необходимую чтобы совпасть с детальной картиной.

```
Черновик         +    Обученная коррекция    =    Точный результат
(цены BS)             (остаток DeepONet)          (качество Хестона)
  дёшево                  обучаемо                    точно
```

Это невероятно эффективно: нужны тысячи черновиков (дёшево) но только сотни детальных картин (дорого) для обучения коррекции.

## Приложение для криптотрейдинга

### Аналогия с рыночным диджеем

Представьте крипторынок как музыкальное выступление:

- **Входная функция**: Последние 60 часов "музыки" цены BTC/USDT (OHLCV данные) -- песня, которая играла
- **Точка запроса**: Будущий момент времени (через 1 час, 4 часа, 24 часа)
- **Выход**: Ожидаемое движение цены в этот момент -- какая нота будет следующей

Branch-сеть (CNN) слушает недавнюю "музыку" и понимает текущий "настрой" (тренд, волатильность, моментум). Trunk-сеть определяет, что происходит на разных временных отступах. Вместе они предсказывают полную "мелодию" будущих цен.

## DeepONet vs другие подходы

| Подход | Аналогия | Ограничение |
|--------|----------|-------------|
| Стандартный MLP | Заучивание одного рецепта | Работает только для этого блюда |
| Рекуррентная НС | Чтение книги слово за словом | Может обрабатывать только последовательности |
| FNO | Музыкальный эквалайзер (частотная область) | Нужны данные на регулярной сетке |
| **DeepONet** | **Обучение кулинарии** | **Справляется с любым блюдом и тарелкой** |

## Ключевые выводы

1. **DeepONet обучает операторы** (отображения функция-в-функцию), а не просто значения функций
2. **Две сети сотрудничают**: Branch кодирует "что" (входную функцию), Trunk кодирует "где" (точку запроса)
3. **Одна модель для всех сценариев**: Не нужно переобучение при изменении рыночных условий
4. **Можно добавить физику**: PI-DeepONet обеспечивает соблюдение финансовых законов
5. **Многоточностность экономит вычисления**: Сочетание дешёвых аппроксимаций с дорогой точностью
6. **Перенос естественен**: Оператор, обученный на акциях, переносится на крипто с минимальной дообучкой

## Простой пример кода

```python
# Простейший возможный DeepONet
import torch
import torch.nn as nn

class SimpleDeepONet(nn.Module):
    def __init__(self):
        super().__init__()
        # Branch: читает 100 ценовых наблюдений → 64 признака
        self.branch = nn.Sequential(
            nn.Linear(100, 128), nn.ReLU(),
            nn.Linear(128, 64)
        )
        # Trunk: читает отступ времени запроса → 64 признака
        self.trunk = nn.Sequential(
            nn.Linear(1, 128), nn.ReLU(),
            nn.Linear(128, 64)
        )

    def forward(self, prices, time_offset):
        b = self.branch(prices)      # Какой это паттерн?
        t = self.trunk(time_offset)   # Где в будущем?
        return (b * t).sum(dim=-1)    # Объединить для предсказания

# Использование
model = SimpleDeepONet()
price_history = torch.randn(32, 100)   # 32 примера, 100 цен каждый
future_time = torch.randn(32, 1)        # Когда предсказывать?
prediction = model(price_history, future_time)
```

Думайте об этом как: **"Исходя из того, что произошло (branch), предсказать, что произойдёт когда (trunk)."**
